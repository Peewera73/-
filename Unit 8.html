หน่วยที่ 8 การเขียนโปรแกรมอย่างง่ายด้วยภาษาซี

หลักการเขียนโปรแกรมภาษาซี
  1. การสร้างโปรแกรม (Source Code)ภาษาซีสามารถใช้โปรแกรมเอดิเตอร์ (เช่น Notepad) เพื่อสร้างโปรแกรมและบันทึกไฟล์ให้มี นามสกุลเป็น .c หรือ .cpp เช่น mysource.cpp

  2. การคอมไพล์โปรแกรม (Compile) เมื่อโปรแกรมถูกสร้างขึ้น ลําดับต่อไปคือการคอมไพล์โปรแกรมเพื่อตรวจสอบข้อผิดพลาด

  3. การเชื่อมโยงโปรแกรม (Link) เมื่อได้ออบเจ็กต์ไฟล์ซึ่งเป็นภาษาเครื่องแล้ว ขั้นตอนต่อไปคือ การลิงก์ด้วยการนําออบเจ็กต์ ไฟล์เชื่อมโยงกับไลบรารี

  4. การสั่งรันโปรแกรม (Run) เมื่อได้เอ็กซีคิวต์ไฟล์แล้ว ไฟล์ดังกล่าวสามารถเรียกใช้งาน (ดับเบิ้ลคลิกที่ชื่อไฟล์) เพื่อสั่งรันหรือ ประมวลผลโปรแกรมได้ทันทำ

กฎเกณฑ์การเขียนโปรแกรมภาษาซี
  1. ที่ส่วนหัวโปรแกรมต้องกําหนดตัวประมวลผล (Preprocessor) ก่อนเสมอ ตัวอย่างเช่น การผนวก เฮดเดอร์ไฟล์อย่าง #include เพื่อใช้งานฟังก์ชันเกี่ยวกับการอินพุตหรือเอาต์พุตข้อมูล เป็นต้น

  2. ชุดคําสั่งต่าง ๆ ใช้อักษรตัวพิมพ์เล็ก

  3. ตัวแปรที่ใช้งานภายในโปรแกรม ต้องถูกประกาศพร้อมระบุชนิดข้อมูลไว้เสมอ

  4. ภายในโปรแกรมต้องมีอย่างน้อยหนึ่งฟังก์ชัน คือฟังก์ชัน main ()

  5. ใช้เครื่องหมาย { (Left Brace) เพื่อบอกจุดเริ่มต้นของชุดคําสั่ง และเครื่องหมาย } เพื่อบอกจุดสิ้นสุด ของชุดคําสั่ง โดยสามารถมีเครื่องหมาย { } (Braces) ซ้อนอยู่ภายในได้

  6. สิ้นสุดของแต่ละประโยคคําสั่ง ต้องจบด้วยเครื่องหมาย ; (Semicolon)เสมอ

  7. สามารถใช้เครื่องหมาย /* comment */ หรือ // comment

อักขระในภาษาซี
โปรแกรมภาษาซีมีกลุ่มอักขระต่าง ๆ ให้ใช้งานที่เรียกว่า Character Sets โดยแบ่งเป็น 2 กลุ่มใหญ่ๆ คือ ชุดอักขระพื้นฐานและชุดอักขระเพื่อการประมวลผล

ชุดอักขระพื้นฐาน (Basic Character Set) แบ่งเป็น 5 กลุ่ม คือ
  1. อักษรตัวพิมพ์ใหญ่ (Alphabets Upper Case) ได้แก่ตัวอักษร A–Z จํานวน 26 ตัว

  2. อักษรตัวพิมพ์เล็ก (Alphabets Lower Case) ได้แก่ตัวอักษร a–z จํานวน 26 ตัว

  3. ตัวเลข (Decimal Digits) ได้แก่ตัวเลข 0–9 จํานวน 10 ตัว

  4. ตัวสัญลักษณ์พิเศษ (Special Symbols) ได้แก่ สัญลักษณ์พิเศษต่าง ๆ จํานวน 29 ตัว

  5. ตัวอักขระที่เป็นช่องว่าง (Whitespace Characters) ได้แก่ ช่องว่างในรูปแบบต่าง ๆ จํานวน 5 ชนิด



ชุดอักขระเพื่อการประมวลผล (Execution Character Set)
ชุดอักขระเพื่อการประมวลผลแบ่งเป็น 2 กลุ่ม คือ

  1. อักขระที่เป็นค่าว่าง (Null Character) อักขระที่เป็นค่าว่าง หรือค่า Null จะใช้สัญลักษณ์ \0 ซึ่งเป็นตัวกําหนดจุดสิ้นสุดของอักษรสตริง (Character String)

  2. อักขระควบคุม (Escape Sequence) เป็นรหัสที่ใช้ควบคุมการแสดงผลทั้งทางจอภาพ และเครื่องพิมพ์ซึ่งกลุ่มของอักขระควบคุมประกอบด้วย



ค่าคงที่ตัวแปรและชนิดข้อมลู และคําเฉพาะ
  1. ค่าคงที่ (Constants) ตัวอักษร ตัวเลข และสัญลักษณ์พิเศษต่าง ๆ เมื่อนํามาประกอบรวมเข้าด้วยกัน จะเกิดคําขึ้นมา ที่สามารถใช้แทนความหมายในรูปของค่าคงที่ ตัวแปรและชนิดข้อมูล

  2. ตัวแปรและชนิดข้อมูลตัวแปรที่ใช้งานในโปรแกรมภาษาซีต้องถูกประกาศขึ้นมาพร้อมระบุชนิดข้อมูล เพื่อให้ทราบว่า ตัวแปรเหล่านั้นจัด



  3. คําเฉพาะ (C Keywords) คําเฉพาะ คือ คําที่ถูกสงวนไว้ในคอมไพเลอร์จึงไม่สามารถนําคําเฉพาะเหล่านี้ไปใช้เพื่อตั้งชื่อ ตัวแปรได้ในภาษาซีมีคําเฉพาะ



นิพจน์และตัวดําเนินการ
คือ นิพจน์ทางคณิตศาสตร์ซึ่งสามารถพบเห็นได้จากสูตรการคํานวณ ตัวเลขต่าง ๆ ดังนั้น นิพจน์จึงประกอบด้วย ตัวแปร ค่าคงที่ และตัวดําเนินการทางคณิตศาสตร์มาประกอบรวมกัน

ตัวดําเนินการในภาษาซีมีหลายชนิด ในที่นี้จะกล่าวถึงตัวดําเนินการที่สําคัญดังต่อไปนี้
  1. ตัวดําเนินการทางคณิตศาสตร์ตัวดําเนินการทางคณิตศาสตร์ในภาษาซีมีอยู่ 5 ตัว คือ



  2. ตัวดําเนินการยูนารี ภาษาซียังมีตัวดําเนินการแบบยูนารีที่นํามาใช้เพื่อสร้างค่าใหม่ขึ้นมา สําหรับตัวดําเนินการยูนารี แบบทั่วไปคือ ตัวดําเนินการยูนารีติดลบ เช่น ค่าตัวเลข 99 เมื่อนําตัวดําเนินการยูนารีติดลบมาใช้จะทําให้ค่า เปลี่ยนไปเป็นค่าติดลบทันทีคือ –99 อย่างไรก็ตาม ตัวดําเนินการยูนารีติดลบดังกล่าว มิใช่ตัวดําเนินการลบแต่ อย่างใด



  3. ตัวดําเนินการเปรียบเทียบและตรรกะ ตัวดําเนินการเปรียบเทียบในภาษาซีประกอบด้วย



  4. ตัวดําเนินการระดับบิต ภาษาซียังรวมดําเนินการระดับบิตในการจัดการกับข้อมูลภายในได้เช่นเดียวกับภาษาระดับต่ํา อย่างแอสเซมบลีซึ่งประกอบด้วย



  5. ตัวดําเนินการกําหนดค่าในภาษาซีมีหลายวิธีด้วยกันในการกําหนดค่าให้กับตัวแปร ซึ่งปกติตัวดําเนินการกําหนด ค่า มักจะใช้เครื่องหมายเท่ากับ = และสามารถนําไปใช้กับการกําหนดสูตรคํานวณต่าง ๆ



  6. ตัวดําเนินการเงื่อนไข ตัวดําเนินการเงื่อนไขจะนํามาใช้เพื่อทดสอบค่านิพจน์ทางตรรกะ ว่าจริงหรือเท็จ โดยมีรูปแบบ ดังนี้



  7. ลําดับความสําคัญของตัวดําเนินการ ตัวดําเนินการแต่ละตัวจะมีลําดับความสําคัญก่อนหลังที่แตกต่างกัน โดยการประมวลผลจะ กระทํากับตัวดําเนินการที่มีลําดับความสําคัญก่อน แต่ถ้ากรณีที่มีลําดับความสําคัญเท่ากัน ตามปกติจะกระทํา กับตัวดําเนินการจากซ้ายไปขวา โดยลําดับความสําคัญของตัวดําเนินการพื้นฐานเป็นดังนี้



ขั้นตอนการติดตั้งโปรแกรม Dev–Cpp เวอร์ชัน 5.4.1
สําหรับการเขียนโปรแกรมภาษาซี นั้นสามารถใช้งานเอดิเตอร์ได้หลายตัว เช่น Notepad, EditPlus, Turbo C++, NetBeans, Visual Studio และ Dev–Cpp เป็นต้น โปรแกรม Dev–Cpp 5.4.1 สามารถนํามา เขียนภาษาซีและ ภาษา C++ ได้ซึ่งโปรแกรมมีลักษณะเด่นคือ มีขนาดเล็กเมื่อเทียบกับโปรแกรม NetBeans, Visual Studio และสําหรับการติดตั้งโปรแกรม Dev–Cpp 5.4.1 มีขั้นตอนการติดตั้งและการใช้งานโปรแกรม ดังต่อไปนี้

  1. เปิดเครื่องคอมพิวเตอร์และเชื่อมต่ออินเทอร์เน็ต จากนั้นเข้าไปที่เว็บเบราร์เซอร์ Google แล้วค้นหา คําว่า “Dev–Cpp 5.4.1 download”



  2. เมื่อดาวน์โหลดเสร็จแล้ว ในที่นี้ให้นําโปรแกรมดังกล่าวจัดเก็บไว้ที่โฟลเดอร์ C:\Dev–Cpp 5.4.1



  3. ทําการแตกไฟล์ด้วยการคลิกขวาที่ชื่อไฟล์แล้วเลือกรายการ Extract Here (ในที่นี้ใช้โปรแกรม WinRAR)



  4. จากนั้นดับเบิลคลิกที่ไฟล์ โปรแกรม Dev–Cpp 5.4.1 MinGW 4.7.2 Setup เพื่อเริ่มการติดตั้ง โปรแกรม



  5. โปรแกรมจะทําการคลายการบีบอัดข้อมูลในการติดตั้งโปรแกรม และแสดงเมนูให้ผู้ใช้เลือกภาษาใน การติดตั้ง โดยทําการเลือกเป็นภาษาอังกฤษ (English) แล้วคลิกปุ่ม OK



  6. โปรแกรมแสดงข้อตกลงในการใช้งานโปรแกรม Dev–C++ 5.4.1 และการยินยอมในการใช้งานตาม ข้อตกลง โดยผู้ใช้ต้องยอมรับข้อตกลงในการใช้งาน คลิกปุ่ม I Agree



  7. เลือกส่วนประกอบในการติดตั้งและใช้งานโปรแกรม โดยผู้ใช้สามารถเลือกการติดตั้งแบบเต็มเวอร์ชัน หรือบางส่วนก็ได้ขึ้นอยู่กับความต้องการใช้งาน แต่ในที่นี้ให้เลือก Full และ คลิกปุ่ม Next > สําหรับการติดตั้ง ขั้นตอนต่อไป



  8. เลือกตําแหน่งในการติดตั้งโปรแกรม Dev–C++ 5.4.1 โดยถ้าไม่มีการเปลี่ยนตําแหน่งในการติดตั้งจะ ทําการติดตั้งตามตําแหน่งดังภาพที่ 8.10 แต่ถ้าต้องการเปลี่ยนตําแหน่งในการติดตั้ง ผู้ใช้งานสามารถกดเลือก ปุ่ม Browse เพื่อเลือกตําแหน่งที่ต้องการติดตั้งได้ถ้าไม่มีการเปลี่ยนตําแหน่งในการติดตั้งให้คลิกปุ่ม Install เพื่อเริ่มการติดตั้งโปรแกรม



  9. โปรแกรมจะดําเนินการติดตั้งโปรแกรม Dev–C++ 5.4.1 และรอจนกว่าการติดตั้งโปรแกรมสําเร็จ



  10. เมื่อโปรแกรมดําเนินการติดตั้งเสร็จเรียบร้อยแล้วจะแสดงหน้าต่างการติดตั้งโปรแกรมที่สมบูรณ์



  11. เมื่อผู้ใช้คลิกปุ่ม Finish จะเป็นการสิ้นสุดการติดตั้งโปรแกรม และเปิดโปรแกรมขึ้นมาใช้งานทันที พร้อมทั้งปรากฏไอคอน Dev–C++ บนเดสก์ท็อป



การเขียนโปรแกรม การตรวจสอบ และการแก้ไขโปรแกรม
เมื่อติดตั้งโปรแกรม Dev–C++ 5.4.1 เสร็จ นั่นหมายความว่า พร้อมที่จะทําการเขียนโปรแกรมได้โดย ในที่นี้ให้ทดลองเขียนโปรแกรมเพื่อแสดงชื่อตนเอง “My name is My name is Mr. Settaphun Suksai”

การเขียนโปรแกรม
  1. ให้ผู้ใช้ดําเนินการเปิดโปรแกรมโดยดับเบิลคลิกไอคอน Dev–C++ บนเดสก์ท็อป

  2. จากนั้นโปรแกรม Dev–C++ 5.4.1 จะเปิดโปรแกรมขึ้นมาให้พร้อมในการใช้งาน



  3. ในการเริ่มต้นการเขียนโปรแกรมนั้นผู้ใช้สามารถเลือกเขียนโปรแกรม Dev–C++ ในรูปแบบ ของ Source File หรือ Project ได้โดยใช้เมาส์คลิกเลือกเมนู File > New



  4. ในกรณีเลือกเขียนโปรแกรมแบบ Source File โปรแกรมจะเปิดไฟล์ชื่อ Untitled1 มาให้ อัตโนมัติและให้เขียนโปรแกรม



  5. จัดเก็บไฟล์คลิกเมนู File > Save AS แล้วคลิกเลือกแหล่งเก็บไฟล์ C:\Coding



การตรวจสอบ
การตรวจสอบโปรแกรมให้คลิกเมนู Execute > Compile หรือ กดแป้นคีย์ F9 เพื่อ คอมไพล์โปรแกรมจะพบว่าเกิดข้อผิดพลาด แจ้งข้อผิดพลาดว่าไม่รู้จัก

  1. การแก้ไขโปรแกรมเมื่อเกิดข้อผิดพลาดโปรแกรมจะไม่สามารถรันได้ดังนั้นต้องเข้าไปแก้ไขคําสั่งในโปรแกรมก่อน ซึ่งจะพบว่าในบรรทัดที่ 4 มีการเขียนคําสั่งผิด ให้แก้ไขจาก P ตัวพิมพ์ใหญ่ มาเป็น p ตัวพิมพ์เล็ก แล้วทําการ ตรวจสอบโปรแกรมอีกครั้ง ซึ่งจะพบว่า ไม่พบข้อผิดพลาด แล้วให้คลิกปุ่ม Close



  2. การรันโปรแกรมสั่งรันโปรแกรมให้คลิกที่ เมนู Execute > Run หรือ กดแป้นคีย์ F10 , หลังจากรันโปรแกรมแล้ว ทดลองเข้าไปยังโฟลเดอร์ C:\Coding จะพบไฟล์ ex01 สกุลต่าง ๆ โดยเฉพาะไฟล์ ex01.exe (Application) ซึ่งเป็นเอ็กซีคิวต์ไฟล์ที่สามารถดับเบิลคลิกเพื่อสั่งรันโปรแกรมดังกล่าวได้ทันที







ชนิดของข้อผิดพลาด
  1. ข้อผิดพลาดที่เกิดจากไวยากรณ์ (Syntax Errors) ข้อผิดพลาดชนิดนี้เกิดจากการใช้ไวยากรณ์หรือรูปแบบภาษาที่ผิด เช่น สะกดคําสั่งผิดแทนที่ จะต้องพิมพ์คําสั่งว่า printf ก็เป็น Printf เป็นต้น ซึ่งเมื่อผ่านการแปลแล้ว ตัวแปลภาษาจะไม่รู้จักคําสั่งดังกล่าว อย่างไรก็ตามข้อผิดพลาดที่เกิดจากไวยากรณ์นั้น ตรวจสอบได้ง่าย เนื่องจากคอมไพเลอร์สามารถตรวจพบและ แจ้งข้อผิดพลาดที่เกิดขึ้นให้รับทราบทันที

  2. ข้อผิดพลาดที่เกิดจากตรรกะโปรแกรม (Logic Errors) ข้อผิดพลาดชนิดนี้จัดเป็นข้อผิดพลาดที่เกิดจากโปรแกรมเมอร์เช่น การใช้ตรรกะในการสร้าง เงื่อนไขที่ผิดพลาดหรือการสร้างสูตรคํานวณที่ผิด ส่งผลให้ผลลัพธ์ผิดพลาดไม่ตรงกับความเป็นจริง เช่น เงื่อนไข ที่ต้องการสร้าง คือ ถ้าคะแนนรวม >= 80 จะได้เกรด A แต่กลับสร้างเงื่อนไขเป็น ถ้าคะแนน > 80 จะได้เกรด A จึงส่งผลให้นักศึกษาที่มีคะแนน 80 ไม่ได้เกรด A ที่สําคัญ ข้อผิดพลาดชนิดนี้คอมไพเลอร์จะไม่สามารถตรวจ พบได้เพราะชุดคําสั่งไม่ได้เขียนผิดรูปแบบแต่อย่างใด ดังนั้น สูตรคํานวณต่าง ๆ จําเป็นต้องได้รับการทดสอบ ความถูกต้องผ่านการป้อนข้อมูลในรูปแบบต่าง ๆ ซ้ำแล้วซ้ำเล่า จนมั่นใจว่าได้ผลลัพธ์ที่ถูกต้อง

  3. ข้อผิดพลาดในขณะรันโปรแกรม (Runtime Errors) สําหรับข้อผิดพลาดแบบ Runtime นั้น คอมไพเลอร์จะไม่สามารถตรวจพบได้เนื่องจากรูปแบบ คําสั่งไม่ได้ผิดพลาดแต่อย่างใด ครั้นเมื่อข้อผิดพลาดชนิดนี้ถูกตรวจพบ โปรแกรมจะหยุดการทํางานทันที ตัวอย่างข้อผิดพลาดชนิดนี้เช่น กรณีสูตรคํานวณที่สร้างขึ้น บังเอิญว่ามีตัวหารเป็นศูนย์ซึ่งโดยปกติแล้วค่าใดๆ ก็ตามเมื่อหารด้วยศูนย์ผลลัพธ์ที่ได้คือ ค่าอนันต์ดังนั้น วิธีป้องกันข้อผิดพลาดจากกรณีนี้คือ ควรเขียน โปรแกรมสร้างเงื่อนไขเพื่อตรวจสอบค่าที่นํามาใช้เป็นตัวหาร ถ้ามีค่าเท่ากับศูนย์จะไม่นํามาคํานวณ แล้วแจ้ง ข้อความให้ผู้ใช้รับทราบทางจอภาพ

สรุปหน่วยที่ 8
หลักการเขียนโปรแกรมภาษาซี ประกอบด้วยขั้นตอนต่อไปนี้
  1. การสร้างโปรแกรม (Source Code)

  2. การคอมไพล์โปรแกรม (Compile)

  3. การเชื่อมโยงโปรแกรม (Link)

  4. การสั่งรันโปรแกรม (Run)

กฎเกณฑ์การเขียนโปรแกรมภาษาซี
  1. ที่ส่วนหัวโปรแกรมต้องกําหนดตัวประมวลผล (Preprocessor) ก่อนเสมอ

  2. ชุดคําสั่งต่าง ๆ ใช้อักษรตัวพิมพ์เล็ก

  3. ตัวแปรที่ใช้งานภายในโปรแกรม ต้องถูกประกาศพร้อมระบุชนิดข้อมูลไว้เสมอ

  4. ภายในโปรแกรมต้องมีอย่างน้อยหนึ่งฟังก์ชัน ก็คือฟังก์ชัน main ()

  5. ใช้เครื่องหมาย { เพื่อบอกจุดเริ่มต้นของชุดคําสั่ง และเครื่องหมาย } เพื่อบอกจุดสิ้นสุดของชุดคําสั่ง โดยสามารถมีเครื่องหมาย { } ซ้อนอยู่ภายในได้

  6. สิ้นสุดของแต่ละประโยคคําสั่ง ต้องจบด้วยเครื่องหมาย ; เสมอ

  7. สามารถใช้เครื่องหมาย /* comment */ หรือ // comment เพื่อระบุหมายเหตุ ภายในโปรแกรม โดยส่วนนี้จะถูกนําไปประมวลผล

อักขระในภาษาซี
  1. ชุดอักขระพื้นฐาน แบ่งเป็น อักษรตัวพิมพ์ใหญ่อักษรตัวพิมพ์เล็ก ตัวเลข ตัวสัญลักษณ์พิเศษ และตัว อักขระที่เป็นช่องว่าง

  2. ชุดอักขระเพื่อการประมวลผล แบ่งเป็น อักขระที่เป็นค่าว่าง และ อักขระควบคุม ค่าคงที่ คือ ข้อมูลจะคงอยู่ตลอดการทํางานของโปรแกรม ไม่สามารถเปลี่ยนแปลงค่าได้

กฎการตั้งชื่อตัวแปร
  1. ชื่อตัวแปร สามารถตั้งชื่อได้ด้วยการนําตัวอักขระ ตัวเลข รวมถึงการนําเครื่องหมายอันเดอร์สกอร์ (Underscore) มาใช้แต่มีข้อแม้ว่า ตัวอักขระตัวแรกต้องเป็นตัวอักษรเท่านั้น อย่างไรก็ตามเครื่องหมาย อันเดอร์สกอร์ (Underscore) จะถือว่าเป็นตัวอักษรตัวหน่งึ จึงสามารถนํามาตั้งชื่อตัวแปรตัวแรกได้

  2. ตัวอักษรตัวพิมพ์ใหญ่และตัวอักษรตัวพิมพ์เล็ก ในภาษาซีถือว่ามีความแตกต่างกัน

  3. ชื่อตัวแปรต้องไม่ตรงกับคําเฉพาะ

  4. ชื่อตัวแปรสามารถมีความยาวได้ 31 ตัวอักษร ตามมาตรฐาน ANSI C

  5. ชื่อตัวแปรไม่อนุญาตให้มีช่องว่าง แต่ถ้าต้องการแยกคํา สามารถใช้เครื่องหมายอันเดอร์สกอร์ (Underscore) ช่วยได้

  6. ควรตตั้งชื่อตัวแปรให้สื่อความหมาย เพื่อคาดเดาได้ว่าชื่อตัวแปรดังกล่าวนําไปใช้เพื่อการใด

คําเฉพาะ
คือ คําที่ถูกสงวนไว้ใช้งานในคอมไพเลอร์ไม่สามารถนําคําเฉพาะเหล่านั้นไปใช้เพื่อตั้งชื่อ ตัวแปรได้

นิพจน์และตัวดําเนินการ
นิพจน์ในภาษาซีสามารถพบเห็นได้จากสูตรการคํานวณตัวเลขต่าง ๆ และ ตัว ดําเนินการในภาษาซีที่สําคัญ เช่น ตัว ดําเนินการทางคณิตศาสตร์ตัวดําเนินการยูนารีตัวดําเนินการ เปรียบเทียบตรรกะ ตัวดําเนินการระดับบิต ตัวดําเนินการกําหนดค่า และตัวดําเนินการเงื่อนไข

โปรแกรม Dev–Cpp เวอร์ชัน 5.4.1
สามารถนํามาเขียนภาษาซีและ ภาษา C++ ได้ซึ่งโปรแกรม มีลักษณะเด่นคือ มีขนาดเล็กเมื่อเทียบกับโปรแกรม NetBeans, Visual ทํางานรวดเร็ว มีประสิทธิภาพ เหมาะ กับการนํามาใช้เพื่อฝึกหัดเขียนโปรแกรมได้เป็นอย่างดีเนื่องจากติดตั้งง่าย ใช้งานสะดวก สามารถหา ดาวน์โหลดมาใช้งานได้จากอินเทอร์เน็ต
